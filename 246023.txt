22)Настана конфликт при merge-от на master во feature-borrow-book.
Конфликтот беше разрешен така што беше отстранета линијата код: Book checked out.
Како и маркерите: <<<<<<< HEAD и >>>>>>> master
Односно одбрав да ја задржам само верзијата од feature-borrow-book branch-от.
За излез од Vim едиторот искористив: Esc -> :wq -> Enter.

Употребена беше merge стратегијата: Recursive.

Излез од командата git status:

nothing to commit, working tree clean

23)Спојувањето беше: Recursive.
Fast-forward спојувањето се случува само ако тековната гранка нема commits што недостасуваат во гранката 
што се спојува. Но во случајот, master веќе имал commits што не биле на feature-genre-report, па Git не 
може едноставно да го помести покажувачот master напред. Наместо тоа, Git креирал commit за спојување 
за да ги комбинира историите на master и feature-genre-report.
